Visualization

In Kibana, you can create objects that describe what data will be displayed in some graph. In terms of Kibana, this is called visualization. Visualizations are stored in underlaying ElasticSearch database as documents that could be referenced from dashboards and displayed. In the following examples, visualizations with various graphs will be created and explained.

Basic example

To get the glimpse of how to create visualization, let start with some basic example of histogram (bar) chart displaying count of documents as time progresses. Lets go through the main points in the code.

Namespace clibana.visualization is plugged in, as it defines functions to build up visualization representation.

With visualization we say that we want to build visualization. It has single mandatory parameter that defines id of the visualization. (This id could be later used to reference visualization from dashboard.) Additional parameters of the visualization are constructed using specific builders that are following.

Next, using with-bar-chart, we say that we want visualization that contains bar chart.

Now, we need to define X and Y axes of the bar chart.

Using chart-with-Y-aggregation, having parameter :count, we say that we want Y axis to be count of documents. There is no additional parameter as count is just count, different Y axis types have extra parameters, but more on this later.

Using chart-with-X-aggregation we define what we want X axis to be. Here we have parameter :date-histogram causing X axis to be made of time ticks (with automatic sizing by default). Next parameter pair :field "@timestamp" define which field is taken as source for those time ticks.

Last, we need to slurp some data. This is done using with-saved-search that references some saved search by its id. We could use here explicit search query, but this will be explained in separate example.
